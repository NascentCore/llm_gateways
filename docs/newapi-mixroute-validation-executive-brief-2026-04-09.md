# api.mixroute.ai验证简报

**验证日期**：2026-04-09  
**说明**：下列结论来自在指定云主机上、使用标准脚本对网关 `https://api.mixroute.ai/v1` 的拨测；反映当时网络、密钥与时段下的观测结果，**不等同于**供应商长期服务等级承诺（SLA）。

---

## 一、我们验证了什么

确认该网关能否被正常调用、多种模型是否可用，并在一种「大提示词 + 长回答」的压力形态下，观察**每分钟能完成多少轮对话**以及据此估算的**每分钟处理多少文本量（token，估算）**。  
通俗理解：**RPM** = 每分钟大约完成多少次对话请求；**TPM** = 每分钟大约处理多少 token（下文说明估算方式与局限）。

---

## 二、连通性结论（能否用上）

在两种企业里常见的对接方式下，下列四个模型**均能成功完成对话**（技术版中为 OpenAI 兼容对话接口与 Anthropic Messages 接口，各测一遍）：

| 模型（名称与验证时一致） |
| ------------------------ |
| claude-opus-4-6          |
| claude-sonnet-4-6        |
| gemini-3-flash-preview   |
| gemini-3.1-pro-preview   |

**业务表述**：网关对「像 OpenAI 那样调」和「像 Anthropic 那样调」两种方式都可用，上述模型在验证环境下**全部通过**，未发现「连不上、调不通」类问题。

---

## 三、性能与容量（RPM 与 TPM 估算）

本节数字来自技术报告中**高并发大负载**一次实测汇总：模型为 **gemini-3-flash-preview**，单次请求按脚本设计约为 **约 1 万输入 token** 与 **最多 1000 输出 token** 的规模；最高并行约 **512** 路；该轮次内 **992 次请求全部成功**，总耗时约 **564 秒**，过程中**未观察到 429 限流**（技术版备注：本次密钥与时段下）。

### 3.1 每分钟请求数（RPM）

按「成功次数 ÷ 总耗时（秒）× 60」计算：

**RPM** = 992 ÷ 563.9005 × 60 ≈ **105.5**（次/分钟）。

**结论**：约 **105.5 次请求/分钟**（与技术报告中的 `approx_throughput_rpm=105.55` 一致）。

### 3.2 每分钟 token 量（TPM，估算）

在**假设**单次请求规模相当于 **10 000 + 1 000 = 11 000** token（与测试脚本设定的输入目标与输出上限相对应）的前提下：

**TPM（估算）** = RPM × (10 000 + 1 000) ≈ 105.55 × 11 000 ≈ **1 161 050**。

**结论**：约 **116 万 token/分钟**（约 **116.1 万 TPM**）。

**重要说明（请务必读）**：这里的 TPM 是**按测试设定的单次请求规模推算的估算值**，不是账单或网关后台的精确统计。真实 token 会随模型实际写出多长、网关如何统计（例如是否含模板/思考等）而变化；若用于成本或配额谈判，应以**官方计量与账单**为准。

## 四、响应时间（快慢大致是什么水平）

同一高并发实测中，单次请求从发起到返回的耗时（墙钟）大致为：

| 指标     | 数值（秒） | 通俗理解         |
| -------- | ---------- | ---------------- |
| 最快     | 约 6.7     | 部分请求较快完成 |
| 一半请求 | 约 11.5    | 半数请求不高于此 |
| 最慢     | 约 318     | 存在明显「长尾」 |

**业务表述**：高并行时会出现少数**特别慢**的请求，多与排队、资源争抢有关，**不代表**日常只开少量对话时的典型体验。

## 五、补充对照（可选阅读）

技术报告中还有一次**较低并行**、同样「大负载」形态的测试，得到的吞吐量约为 **21.6 次请求/分钟**。与上文 **105.5 次/分钟** 的场景**不能直接对比业务目标**，因为并发阶梯与总耗时不同；仅说明「压法不同，数字会差很多」。

## 六、风险与后续建议

1. **密钥安全**：技术记录注明 API Key 曾在聊天中暴露，**建议在 NewAPI 后台尽快轮换**，避免未授权使用。
2. **指标口径**：本简报的 RPM/TPM 来自「整段测试时长内的平均吞吐」类口径；技术报告中另有「以首次失败时刻为锚的 60 秒窗口」等**不同定义**，二者不可混为一谈。
3. **环境单一**：验证在固定云主机上完成，若生产环境网络或地域不同，表现可能略有差异。
